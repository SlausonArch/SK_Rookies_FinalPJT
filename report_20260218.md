# 2026-02-18 작업 보고서

## 1. 개요
본 보고서는 금일 진행된 거래소(`Exchange.tsx`) 페이지 오류 수정 및 투자 내역(`Investments.tsx`) 페이지의 "오늘의 수익" 계산 로직 개선 작업에 대한 내역을 정리한 것입니다.

## 2. 주요 수정 내역

### 2.1 거래소 페이지 (`Exchange.tsx`)
- **문제점**: 페이지 접속 시 흰 화면이 뜨며 `ReferenceError: PageContainer is not defined` 오류 발생. 또한 스타일 컴포넌트 태그 불일치(`<PageContainer> ... </ExchangeContainer>`)로 인한 JSX 파싱 에러 발생.
- **원인**: 이전 코드 통합 과정에서 스타일 컴포넌트 정의가 누락되거나, 여는 태그와 닫는 태그의 이름이 일치하지 않는 실수가 발생함.
- **해결**:
  - 누락된 `PageContainer`를 `ExchangeContainer`로 재정의하여 추가.
  - 여는 태그와 닫는 태그를 `<ExchangeContainer>`로 통일.
  - `useEffect` 훅 복구: URL 파라미터(`market`)에 따른 코인 선택 및 마켓 목록 조회 로직 복구.
  - 관련 `styled-components` 정의 복구 (`ExchangeBody`, `SidebarPanel`, `ChartPanel`, `OrderbookPanel`, `RecentTradesPanel`, `MarketInfo` 등).

### 2.2 투자 관리 페이지 (`Investments.tsx`)
- **문제점**: "오늘의 수익"이 단순 `매도금액 - 매수금액`으로 계산되어, 매수만 수행한 경우 수익이 마이너스로 표시되는 등 실제 평가 손익을 반영하지 못함.
- **해결**:
  - 계산 로직 변경: **(보유 코인 수량 × 전일 대비 등락액)** 합계로 수정.
  - Upbit API의 `signed_change_price`(전일 대비 값 변화량) 데이터를 활용하여 실시간 평가 손익을 반영.
  - `coinPrices` 상태 객체 구조 변경: 단순 가격(`number`)에서 `{ price: number, changePrice: number }` 객체 형태로 확장하여 등락액 정보 포함.

### 2.3 공통 사항
- **Lint 오류 수정**: 사용하지 않는 변수 제거 및 타입 정의 보완.
