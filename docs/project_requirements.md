# LLM 기반 보안 로그 분석 및 시각화 프로젝트 요구사항 분석서

## 1. 프로젝트 개요 (Project Overview)

본 프로젝트는 **'LLM 기반 보안 로그 분석 및 시각화'**를 주제로 하며, 보안 취약점 진단 교육 및 실습을 위한 모의 가상자산 거래소 플랫폼을 구축하는 것을 목표로 합니다.

- **프로젝트 기간**: 4일 (단기 프로젝트)
- **핵심 목표**: 데이터 분석 및 시각화를 통한 인사이트 도출 (자동화 기능은 범위 제외)
- **핵심 흐름**: `로그(Logs)` -> `LLM 분석` -> `JSON 변환` -> `Streamlit 시각화`
- **개발 목적**: 실제 금융 서비스 수준의 보안이 아닌, **취약점 진단 교육용** 환경 제공

## 4. 개발 우선순위 및 담당자 (Development Roadmap)

### **P0 (Critical - 필수 구현)**
서비스의 핵심이 되는 기능으로, 가장 먼저 개발되어야 합니다.

- **회원가입 / 로그인** (담당: **김한수**)
    - **페이지**: 회원가입 폼, 로그인 페이지. (로그아웃은 버튼만 구현)
    - **기능**:
        - 소셜 로그인 연동.
        - **KYC 정보**: 이름, 전화번호, 주소, 계좌번호, 주민등록번호(뒷자리 1자리까지), 신분증 사진 URL.
    - **DB**: 회원 정보 테이블 설계 및 구현.

- **거래소 페이지** (담당: **박지빈**)
    - **디자인**: 업비트(Upbit) UI 벤치마킹.
    - **기능**: 오더북(호가창), 차트 영역, 매수/매도 주문창.

- **초기 DB 설계 및 구현** (담당: **박찬웅**)
    - **개인 자산**: 보유 코인, 예치금 등.
    - **커뮤니티**: 게시글, 댓글, 좋아요, 관리자 필터링, 검색 기능.
        - *Feature*: 공지사항 우선 정렬 로직.
    - **수수료 티어**: 거래 총량 트리거에 따른 등급/수수료율 테이블.
    - **회원 정보**: 위 KYC 정보 포함 테이블.

### **P1 (Major - 중요 구현)**
핵심 기능 이후 사용자 경험을 위해 필요한 기능입니다.

- **메인 페이지** (담당: **김하늘**)
    - **기능**: '거래소 둘러보기' 버튼, 핫한 코인 목록 리스트, 공지사항 배너/리스트 노출, 헤더 포함.

- **커뮤니티 페이지** (담당: **김동현**)
    - **일반 사용자**: 글쓰기(사진 업로드), 수정(본인 글만), 삭제, 좋아요, 댓글 작성.
    - **관리자**: 공지사항 작성, 모든 글 수정/삭제/사용자 제재/댓글 삭제.

- **관리자 페이지** (담당: **P1**)
    - 사용자 관리, 공지사항 관리 등 운영 기능.

### **P2 (Minor - 후순위 구현)**
시간적 여유가 있을 때 구현합니다.

- **내 자산 보기**:
    - 보유 자산 현황, 투자 손익(PnL), 거래 내역.
    - 미체결 주문, 입출금 대기 내역, 자동 주문 현황.

- **마이페이지 세부 기능**:
    - 닉네임 변경, 회원정보 수정, 입출금 계좌 관리.
    - 2채널 인증(2FA), 개인 지갑 주소 관리.
    - 개인 수수료율 조회.
- **기타**: 버튼 구성 디테일 등 UI 폴리싱.

## 5. 상세 기능 요구사항 및 진단 포인트 (Functional Details & Auditing Points)

### 5.1 계정 및 인증
- **회원 등급제**: 거래량/예치금에 따른 등급(Bronze, Silver, Gold, VIP) 및 수수료 차등.
- **진단 포인트**: 파일 업로드(WebShell), 파라미터 변조, IDOR.

### 5.2 거래 및 금융
- **수수료 정책**: Bronze(0.08%), Silver(0.05%), Gold(0.03%), VIP(0.01%).
- **입출금**: 외부 지갑 송금 및 가상 계좌 입금.
- **진단 포인트**: Rounding Error, 마이너스 금액 입력, Race Condition.

### 5.3 커뮤니티 및 정보
- **게시판/이벤트**: 추천인 코드, 신규 가입 에어드랍.
- **진단 포인트**: XSS, CSRF, SSRF.

### 5.4 관리자 / 운영 도구
- **기능**: 사용자 검색/제재, 거래 모니터링, 수수료 정책 설정.
- **진단 포인트**: 권한 상승, 로그 위·변조, 관리자 API 오용.

## 6. 비기능 요구사항 (Non-Functional Requirements)

### 6.1 보안 및 목적
- **목적**: 취약점 진단 교육용 (실제 금융 보안 수준 아님).
- **의도적 취약점 정책**:
    - 취약점은 통제된 범위 내에서 의도적으로 잔존시킴.
    - 실제 개인정보 사용 금지, 더미 데이터 사용.
    - 관리자 페이지는 별도 경로/IP 제한 등 망 분리 적용.

### 6.2 데이터 및 로깅
- **데이터**: 전수 더미 데이터 사용 (신분증 이미지 포함).
- **로깅**: 진단을 위한 요청/응답 메타데이터, 관리자 행동, 핵심 이벤트 로그 필수.
- **복구**: 장애 발생 시 30분 내 복구 가능한 초기화 스크립트/스냅샷 보유.
